# 大学生就业信息管理系统需求文档

## 1. 项目概述

### 1.1 项目背景

当前大学生就业市场信息分散，学生获取精准就业信息难度大，学校也难以高效管理就业数据和对接企业需求。基于Spring Boot框架构建的大学生就业信息管理系统，旨在解决这些问题，为学生、企业和学校提供一个高效、便捷的就业服务平台。

### 1.2 项目目标

- 为学生提供集中、便捷的就业信息获取平台
- 实现就业数据的统一管理，方便学校及时掌握学生就业情况
- 帮助企业更精准地推送招聘信息，吸引合适的大学生人才
- 提升就业管理效率，优化就业服务流程

## 2. 功能需求

### 2.1 角色管理

- **学生角色**：注册、登录、个人信息管理、简历管理、岗位搜索、简历投递、岗位收藏、求职反馈
- **企业角色**：注册、登录、企业信息管理、岗位发布、简历筛选、招聘响应、企业评价
- **学校管理员角色**：系统管理、用户管理、数据统计、信息审核、校企对接管理

### 2.2 核心功能模块

#### 2.2.1 用户管理模块

- 用户注册（学生、企业、管理员）
- 用户登录与身份验证
- 个人信息管理与修改
- 密码重置功能

#### 2.2.2 岗位信息模块

- 岗位发布与管理（企业）
- 岗位分类与标签管理
- 岗位搜索与筛选（按行业、薪资、地点、专业等）
- 岗位详情查看
- 岗位收藏功能

#### 2.2.3 简历管理模块

- 简历创建与编辑（学生）
- 简历模板选择
- 简历预览与导出
- 简历投递记录管理

#### 2.2.4 招聘流程模块

- 简历投递功能
- 企业简历筛选与回复
- 面试通知与管理
- 录用管理

#### 2.2.5 数据统计模块

- 就业数据统计（学校）
- 岗位需求分析
- 学生就业情况分析
- 企业招聘效果分析

#### 2.2.6 评价反馈模块

- 学生对企业的评价
- 企业对学生的评价
- 就业服务满意度评价

## 3. 非功能需求

### 3.1 性能需求

- 高并发处理能力：支持多人同时查询岗位、投递简历
- 快速响应时间：页面加载时间不超过3秒
- 系统稳定性：7×24小时稳定运行

### 3.2 安全性需求

- 用户数据加密存储
- 权限控制与访问管理
- 防止SQL注入、XSS等安全攻击
- 敏感信息保护

### 3.3 可用性需求

- 系统可用性达到99.9%
- 故障恢复机制
- 数据备份与恢复

### 3.4 可扩展性需求

- 模块化设计，便于功能扩展
- 支持未来业务需求的增长
- 易于集成第三方服务

## 4. 技术选型

### 4.1 后端技术

- **框架**：Spring Boot 3.0+
- **数据库**：MySQL 8.0+
- **持久层**：MyBatis-Plus
- **缓存**：Redis
- **认证**：JWT
- **构建工具**：Maven

### 4.2 前端技术

- **框架**：Vue 3
- **构建工具**：Vite
- **UI组件**：Element Plus
- **状态管理**：Pinia
- **路由**：Vue Router
- **HTTP客户端**：Axios

## 5. 系统架构

### 5.1 整体架构

采用前后端分离架构，具体包括：

- **前端应用层**：Vue 3 + Element Plus
- **后端服务层**：Spring Boot + Spring Security
- **数据存储层**：MySQL + Redis

### 5.2 核心模块划分

| 模块名称     | 主要职责                     | 技术实现                   |
| ------------ | ---------------------------- | -------------------------- |
| 用户认证模块 | 用户注册、登录、权限管理     | Spring Security + JWT      |
| 岗位管理模块 | 岗位发布、搜索、筛选         | Spring Boot + MyBatis-Plus |
| 简历管理模块 | 简历创建、编辑、投递         | Spring Boot + 文件存储     |
| 招聘流程模块 | 简历筛选、面试通知、录用管理 | Spring Boot + 消息队列     |
| 数据统计模块 | 就业数据统计、分析           | Spring Boot + ECharts      |
| 评价反馈模块 | 企业评价、学生评价           | Spring Boot + 数据存储     |

## 6. 数据库设计

### 6.1 核心表结构

#### 6.1.1 用户表（user）

- id: 主键
- username: 用户名
- password: 密码（加密存储）
- role: 角色（student/company/admin）
- status: 状态（active/inactive）
- create_time: 创建时间
- update_time: 更新时间

#### 6.1.2 学生表（student）

- id: 主键
- user_id: 关联用户表
- name: 姓名
- gender: 性别
- student_id: 学号
- major: 专业
- grade: 年级
- phone: 手机号
- email: 邮箱
- avatar: 头像

#### 6.1.3 企业表（company）

- id: 主键
- user_id: 关联用户表
- company_name: 企业名称
- industry: 行业
- scale: 规模
- address: 地址
- contact_name: 联系人
- contact_phone: 联系电话
- contact_email: 联系邮箱
- logo: 企业logo
- status: 审核状态

#### 6.1.4 岗位表（job）

- id: 主键
- company_id: 关联企业表
- job_title: 职位名称
- salary_min: 最低薪资
- salary_max: 最高薪资
- location: 工作地点
- education: 学历要求
- experience: 经验要求
- industry: 行业
- description: 职位描述
- requirements: 职位要求
- status: 状态（active/expired）
- publish_time: 发布时间

#### 6.1.5 简历表（resume）

- id: 主键
- student_id: 关联学生表
- resume_name: 简历名称
- basic_info: 基本信息（JSON）
- education: 教育背景（JSON）
- experience: 工作经历（JSON）
- projects: 项目经验（JSON）
- skills: 技能证书（JSON）
- status: 状态
- create_time: 创建时间

#### 6.1.6 投递记录表（application）

- id: 主键
- student_id: 关联学生表
- job_id: 关联岗位表
- resume_id: 关联简历表
- status: 状态（pending/reviewed/interview/offered/rejected）
- create_time: 投递时间
- update_time: 更新时间

#### 6.1.7 评价表（evaluation）

- id: 主键
- evaluator_id: 评价人ID
- evaluatee_id: 被评价人ID
- type: 评价类型（company/student）
- score: 评分
- content: 评价内容
- create_time: 创建时间

## 7. API设计

### 7.1 用户相关API

- POST /api/auth/register - 用户注册
- POST /api/auth/login - 用户登录
- GET /api/auth/info - 获取用户信息
- PUT /api/auth/info - 更新用户信息
- PUT /api/auth/password - 修改密码

### 7.2 岗位相关API

- POST /api/jobs - 发布岗位（企业）
- GET /api/jobs - 获取岗位列表
- GET /api/jobs/{id} - 获取岗位详情
- PUT /api/jobs/{id} - 更新岗位信息
- DELETE /api/jobs/{id} - 删除岗位
- GET /api/jobs/search - 搜索岗位

### 7.3 简历相关API

- POST /api/resumes - 创建简历
- GET /api/resumes - 获取简历列表
- GET /api/resumes/{id} - 获取简历详情
- PUT /api/resumes/{id} - 更新简历
- DELETE /api/resumes/{id} - 删除简历
- POST /api/resumes/{id}/preview - 预览简历

### 7.4 投递相关API

- POST /api/applications - 投递简历
- GET /api/applications - 获取投递记录
- PUT /api/applications/{id}/status - 更新投递状态

### 7.5 统计相关API

- GET /api/statistics/employment - 就业统计
- GET /api/statistics/recruitment - 招聘统计

### 7.6 评价相关API

- POST /api/evaluations - 提交评价
- GET /api/evaluations - 获取评价列表

## 11. 风险评估

### 11.1 技术风险

- **风险**：Spring Boot和Vue 3版本兼容性问题
- **应对措施**：选择稳定版本，进行充分测试

### 11.2 业务风险

- **风险**：企业信息审核不严格
- **应对措施**：建立严格的审核流程，人工审核与系统验证相结合

### 11.3 数据风险

- **风险**：数据丢失或泄露
- **应对措施**：定期备份数据，加密存储敏感信息，建立数据访问控制

### 11.4 性能风险

- **风险**：高并发场景下系统性能下降
- **应对措施**：优化数据库查询，使用缓存，负载均衡

## 12. 结论

基于Spring Boot的大学生就业信息管理系统，通过整合学生、企业和学校三方需求，构建了一个高效、便捷的就业服务平台。系统采用现代化的技术栈，具有良好的性能、安全性和可扩展性。通过本系统的实施，将有效解决大学生就业信息管理中的痛点问题，提升就业服务质量和效率，为大学生就业提供有力的支持。
