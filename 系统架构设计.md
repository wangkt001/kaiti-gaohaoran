# 大学生就业信息管理系统架构设计

## 1. 系统架构

### 1.1 整体架构

采用前后端分离架构，具体包括：

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│                 │     │                 │     │                 │
│   前端应用层     │────>│   后端服务层     │────>│   数据存储层     │
│   (Vue 3)       │<────│   (Spring Boot)  │<────│   (MySQL/Redis) │
│                 │     │                 │     │                 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 1.2 核心模块划分

| 模块名称     | 主要职责                     | 技术实现                   |
| ------------ | ---------------------------- | -------------------------- |
| 用户认证模块 | 用户注册、登录、权限管理     | Spring Security + JWT      |
| 岗位管理模块 | 岗位发布、搜索、筛选         | Spring Boot + MyBatis-Plus |
| 简历管理模块 | 简历创建、编辑、投递         | Spring Boot + 文件存储     |
| 招聘流程模块 | 简历筛选、面试通知、录用管理 | Spring Boot + 消息队列     |
| 数据统计模块 | 就业数据统计、分析           | Spring Boot + ECharts      |
| 评价反馈模块 | 企业评价、学生评价           | Spring Boot + 数据存储     |

## 2. 功能模块设计

### 2.1 用户管理模块

#### 2.1.1 学生用户

- **注册功能**：邮箱/手机号注册，验证码验证
- **登录功能**：账号密码登录，记住密码
- **个人信息管理**：基本信息修改，头像上传
- **密码管理**：密码修改，忘记密码

#### 2.1.2 企业用户

- **注册功能**：企业信息提交，资质审核
- **登录功能**：账号密码登录
- **企业信息管理**：企业资料修改，联系方式更新

#### 2.1.3 管理员用户

- **登录功能**：账号密码登录
- **用户管理**：用户列表查看，权限管理
- **系统设置**：系统参数配置，日志管理

### 2.2 岗位信息模块

#### 2.2.1 岗位发布

- 岗位基本信息填写（职位名称、薪资、地点等）
- 岗位要求设置（学历、专业、经验等）
- 岗位状态管理（发布、下架、过期）

#### 2.2.2 岗位搜索

- 关键词搜索
- 多条件筛选（行业、薪资、地点、专业等）
- 排序功能（最新、薪资、热度等）

#### 2.2.3 岗位详情

- 岗位详细信息展示
- 企业信息展示
- 投递按钮
- 收藏功能

### 2.3 简历管理模块

#### 2.3.1 简历创建

- 基本信息填写
- 教育背景添加
- 工作/实习经历添加
- 项目经验添加
- 技能证书添加

#### 2.3.2 简历编辑

- 在线编辑功能
- 简历模板选择
- 简历预览

#### 2.3.3 简历投递

- 选择简历投递
- 投递记录查看
- 投递状态跟踪

### 2.4 招聘流程模块

#### 2.4.1 简历筛选

- 企业收到的简历列表
- 简历筛选（通过/拒绝）
- 简历标记与备注

#### 2.4.2 面试管理

- 面试通知发送
- 面试时间安排
- 面试结果记录

#### 2.4.3 录用管理

- 录用通知发送
- 录用确认
- 就业状态更新

### 2.5 数据统计模块

#### 2.5.1 就业数据统计

- 就业率统计
- 就业行业分布
- 薪资水平分析
- 就业地区分布

#### 2.5.2 企业招聘分析

- 岗位需求分析
- 招聘效果统计
- 企业满意度调查

### 2.6 评价反馈模块

#### 2.6.1 学生评价企业

- 企业评价提交
- 评价内容管理
- 评价查看

#### 2.6.2 企业评价学生

- 学生评价提交
- 评价内容管理
- 评价查看

## 3. 技术实现方案

### 3.1 后端技术实现

#### 3.1.1 核心依赖

- Spring Boot 3.0+
- Spring Security
- MyBatis-Plus
- Redis
- JWT
- MySQL

#### 3.1.2 目录结构

```
src/
├── main/
│   ├── java/
│   │   └── com/
│   │       └── example/
│   │           └──就业系统/
│   │               ├── controller/    # 控制器
│   │               ├── service/       # 服务层
│   │               ├── mapper/        # 数据访问层
│   │               ├── entity/        # 实体类
│   │               ├── dto/           # 数据传输对象
│   │               ├── config/        # 配置类
│   │               ├── util/          # 工具类
│   │               └── Application.java  # 应用入口
│   └── resources/
│       ├── application.yml  # 配置文件
│       └── mapper/          # XML映射文件
└── test/                    # 测试代码
```

#### 3.2.1 核心依赖

- Vue 3


## 4. 数据库设计

### 4.1 核心表结构

#### 4.1.1 用户表（user）

- id: 主键
- username: 用户名
- password: 密码（加密存储）
- role: 角色（student/company/admin）
- status: 状态（active/inactive）
- create_time: 创建时间
- update_time: 更新时间

#### 4.1.2 学生表（student）

- id: 主键
- user_id: 关联用户表
- name: 姓名
- gender: 性别
- student_id: 学号
- major: 专业
- grade: 年级
- phone: 手机号
- email: 邮箱
- avatar: 头像

#### 4.1.3 企业表（company）

- id: 主键
- user_id: 关联用户表
- company_name: 企业名称
- industry: 行业
- scale: 规模
- address: 地址
- contact_name: 联系人
- contact_phone: 联系电话
- contact_email: 联系邮箱
- logo: 企业logo
- status: 审核状态

#### 4.1.4 岗位表（job）

- id: 主键
- company_id: 关联企业表
- job_title: 职位名称
- salary_min: 最低薪资
- salary_max: 最高薪资
- location: 工作地点
- education: 学历要求
- experience: 经验要求
- industry: 行业
- description: 职位描述
- requirements: 职位要求
- status: 状态（active/expired）
- publish_time: 发布时间

#### 4.1.5 简历表（resume）

- id: 主键
- student_id: 关联学生表
- resume_name: 简历名称
- basic_info: 基本信息（JSON）
- education: 教育背景（JSON）
- experience: 工作经历（JSON）
- projects: 项目经验（JSON）
- skills: 技能证书（JSON）
- status: 状态
- create_time: 创建时间

#### 4.1.6 投递记录表（application）

- id: 主键
- student_id: 关联学生表
- job_id: 关联岗位表
- resume_id: 关联简历表
- status: 状态（pending/reviewed/interview/offered/rejected）
- create_time: 投递时间
- update_time: 更新时间

#### 4.1.7 评价表（evaluation）

- id: 主键
- evaluator_id: 评价人ID
- evaluatee_id: 被评价人ID
- type: 评价类型（company/student）
- score: 评分
- content: 评价内容
- create_time: 创建时间

## 5. API设计

### 5.1 用户相关API

- POST /api/auth/register - 用户注册
- POST /api/auth/login - 用户登录
- GET /api/auth/info - 获取用户信息
- PUT /api/auth/info - 更新用户信息
- PUT /api/auth/password - 修改密码

### 5.2 岗位相关API

- POST /api/jobs - 发布岗位（企业）
- GET /api/jobs - 获取岗位列表
- GET /api/jobs/{id} - 获取岗位详情
- PUT /api/jobs/{id} - 更新岗位信息
- DELETE /api/jobs/{id} - 删除岗位
- GET /api/jobs/search - 搜索岗位

### 5.3 简历相关API

- POST /api/resumes - 创建简历
- GET /api/resumes - 获取简历列表
- GET /api/resumes/{id} - 获取简历详情
- PUT /api/resumes/{id} - 更新简历
- DELETE /api/resumes/{id} - 删除简历
- POST /api/resumes/{id}/preview - 预览简历

### 5.4 投递相关API

- POST /api/applications - 投递简历
- GET /api/applications - 获取投递记录
- PUT /api/applications/{id}/status - 更新投递状态

### 5.5 统计相关API

- GET /api/statistics/employment - 就业统计
- GET /api/statistics/recruitment - 招聘统计

### 5.6 评价相关API

- POST /api/evaluations - 提交评价
- GET /api/evaluations - 获取评价列表
